NOTES

There are several flavors of cubical type theory. One takes its roots from BCH paper that was formally given in CCHM and implemented in Haskell as cubicaltt. The second lineage is cartesian cubical type theory that was presented in ABCFHL and AFH papers, and implemented as redtt, yacctt, and Arend.

Groupoid Infinity:

cubicaltt

The cubicaltt is a cubical type checker by Anders Mörtberg that supports path interval, its connections, homogeneous path compositions, higher inductive types, gluening of types and values. Recursive HITs have complete support only in hcomp version.

  • groupoid/cubical — Groupoid Infinity for cubicaltt.
  • groupoid/hcomp — Hopf Fibrations for cubicaltt.
  • groupoid/infinity — cubical type checker (Erlang).
  • mortberg/cubicaltt — cubical type checker (Haskell).
  • mortberg/yacctt — cartesian cubical type checker (Haskell).

Arend

Arend is a cartesian cubical type checker by JetBrains that supports HITs, extensible records, type classes, implicit variables. It also contains Agda-style reasoning library.

  • groupoid/arend — Groupoid Infinity for Arend.
  • JetBrains/Arend — Arend proof assistant (Java).

Lean

Ground Zero is a cubical model and cubical base library embedded into Lean 3.

  • groupoid/lean — Ground Zero cubical library for Lean 3.
  • gebner/hott3 — HoTT library for Lean 3.
  • leanprover/lean — Lean 3.4.1 (C++).

Other Cubical Type Systems and Libraries:

Agda

This is a base library for Agda in cubical mode.

  • Saizan/cubical-demo — Cubical Demo for Agda --cubical.
  • agda/agda — Agda proof assistant (Haskell).

RedPRL

The redtt is a cartesian cubical type checker that take its roots from RedPRL and JonPRL.

  • RedPRL/RedPRL — PRL proof assistant (Standard ML).
  • RedPRL/redtt — cartesian cubical type checker (OCaml).
